## 🚀 快速让项目运行起来

### 第 1 步：准备媒体文件（5 分钟）
```bash
1. 准备 MP3 歌曲文件，放入: config/music/
2. 准备 PNG/JPG 图片文件，放入: config/images/
   （或使用示例占位符）
```

### 第 2 步：配置卡组（10 分钟）
```bash
编辑 config/decks/deck1.csv，按照 CSV 格式添加卡牌配置
```

### 第 3 步：编译运行（5 分钟）
```bash
# Maven 方式（推荐）
mvn clean javafx:run

# 或者 Java 命令
java -cp . src/Main.java
```

### 第 4 步：享受游戏！🎮
```
1. 选择卡组
2. 设置回合数
3. 点击开始游戏
4. 监听歌曲，按按钮提交结果
```

## 🔍 技术亮点

### 1. 分层架构
```
Model Layer      → 纯数据模型，无业务逻辑依赖
Config Layer     → 灵活的 CSV/TXT 配置
Audio Layer      → 封装音频处理细节  
Game Layer       → 独立的游戏逻辑
UI Layer         → 展示层与业务逻辑分离
```

### 2. 事件驱动设计
```java
gameEngine.setGameListener(new GameEngine.GameListener() {
    @Override
    public void onRoundStart(Card card) { ... }
    @Override
    public void onMusicComplete() { ... }
    @Override
    public void onGameOver(GameState state) { ... }
});
```

### 3. 数据验证机制
```
加载 CSV → 验证头行 → 检查文件存在 
→ 验证格式 → 加载数据 → 报告错误
```

### 4. 实时监控
```
后台面板 → 自动刷新（1秒）→ 显示最新状态 
→ 支持手动调整 → 防止误操作
```

## 🐞 已知限制

### 当前版本的限制
1. **Windows/Mac only** - 需要支持 JavaFX 的系统
2. **单机操作** - 不支持网络多人游戏
3. **本地存储** - 数据不持久化
4. **手动配置** - 需要用户手动编辑 CSV 文件

### 缓解方案
- Phase 2 中将提供 GUI 配置工具
