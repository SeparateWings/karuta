## 系统要求

### 硬件
- **处理器**: Intel Core i3 或同等 AMD 处理器
- **内存**: 2GB RAM 最低，推荐 4GB
- **存储**: 100MB 可用空间
- **显示**: 1024×768 分辨率及以上
- **音频**: 支持 3.5mm 耳机或内置扬声器

### 软件
- **操作系统**: Windows 7/8/10/11 或 macOS 10.15+
- **运行环境**: Java Runtime Environment (JRE) 11 或更高版本
- **其他**: 无需额外软件（JavaFX 已集成）

### 网络
- 首次运行时无需网络连接
- 可选：连接网络进行在线排行榜（未来版本）

## 安装和启动

### 第一次设置

#### 1. 安装 Java（如果尚未安装）
```
Windows:
1. 访问 https://www.java.com/zh_CN/download/
2. 下载 Java Runtime
3. 按照提示完成安装
4. 重启计算机
```

#### 2. 准备应用文件
```
1. 解压 karuta.zip 到任意位置
2. 进入 karuta 文件夹
3. 双击 start.bat（Windows）或 start.sh（Mac/Linux）
```

#### 3. 首次运行检查
- ✅ 应用窗口成功打开
- ✅ 能看到卡组选择界面
- ✅ 工作目录创建完成

### 快速启动

#### Windows 用户
- 双击 `start.bat` 文件

#### Mac/Linux 用户
```bash
./start.sh
```

#### 命令行启动
```bash
java -cp . -jar karuta-jukebox.jar
```

## 基本操作

### 卡组选择界面

```
┌─────────────────────────────────────┐
│   🎵 歌牌点歌系统                    │
│   Karuta Jukebox                   │
├─────────────────────────────────────┤
│ 可用卡组:                           │
│  ☑ 示例卡组 (点击选择)              │
│  ☐ 自定义卡组1                      │
│                                    │
│ 总回合数: [  10  ] (1-100)         │
│                                    │
│  [▶ 开始游戏]  [✕ 退出]            │
└─────────────────────────────────────┘
```

#### 操作步骤
1. **选择卡组** - 从列表中点击一个卡组
   - 蓝色高亮表示已选中
   - 显示卡组名称和文件名

2. **设置回合数** - 使用 +/- 按钮或直接输入
   - 范围: 1-100 回合
   - 推荐: 10-20 回合

3. **开始游戏** - 点击绿色 "▶ 开始游戏" 按钮
   - 系统会检查卡组完整性
   - 加载所有图片和音乐文件
   - 启动游戏界面

4. **退出程序** - 点击红色 "✕ 退出" 按钮

### 游戏进行界面

```
┌────────────────────────────────────────────┐
│ 回合: 3/10 | 成功: 2 失败: 1 成功率: 66.7% │
├────────────────────────────────────────────┤
│                                            │
│               🎴 卡片显示                   │
│             (300×400 像素)                 │
│          【作品名称】                      │
│                                            │
│      🎵 song_name.mp3 播放中...           │
│      [░░░░░░░░░░░░░░░░░░░░] 15秒          │
│                                            │
├────────────────────────────────────────────┤
│   [✓ 对局成功]    [✗ 对局失败]            │
│                                            │
│   [← 返回菜单]                            │
└────────────────────────────────────────────┘
```

#### 游戏操作说明

1. **观看卡片**
   - 左侧大图显示当前卡片
   - 下方显示卡片名称
   - 顶部显示进度信息

2. **监听音乐**
   - 系统自动播放 10-30 秒的歌曲段落
   - 进度条显示播放进度
   - 音量通过系统设置调节

3. **提交结果**（音乐播放完毕后）
   - **✓ 对局成功** - 该卡从场上移除
     - 卡片不会再出现
     - 回合数 +1
     - 成功次数 +1
   
   - **✗ 对局失败** - 该卡保留在场上
     - 卡片可能在后续再次出现
     - 回合数 +1
     - 失败次数 +1

4. **中场休息**
   - 系统每 5 回合自动播放休息音乐
   - 休息界面会出现 **"准备"** 按钮
   - 点击继续下一回合

5. **返回菜单**
   - 点击 **"← 返回菜单"** 即可返回卡组选择
   - 游戏数据会被清除

## 配置数据

### 文件夹结构

```
karuta/
├── config/
│   ├── config.txt              # 主配置
│   ├── music/                  # 🎵 音乐文件
│   │   ├── song1.mp3
│   │   ├── song2.mp3
│   │   └── rest.mp3
│   ├── images/                 # 🖼️ 卡片图片
│   │   ├── card1.png
│   │   ├── card2.jpg
│   │   └── ...
│   └── decks/                  # 卡组配置
│       ├── deck1.csv
│       ├── deck2.csv
│       └── ...
```

### 第一步：添加音乐文件

1. **获取音乐**
   - 准备 MP3, WAV, FLAC, OGG 或 AAC 格式的音乐文件
   - 建议时长: 10-60 秒（会被自动截断）
   - 文件名建议: 英文或拼音（如: song1.mp3, opening.mp3）

2. **放入文件夹**
   ```
   复制到: karuta/config/music/
   示例文件名:
   - spring_theme.mp3
   - summer_opening.mp3
   - rest_music.mp3
   ```

3. **验证**
   - 检查文件是否完整
   - 尝试用媒体播放器打开

### 第二步：添加卡片图片

1. **准备图片**
   - 格式: PNG（推荐）或 JPG
   - 分辨率: 300×400 像素（推荐）
   - 文件名编码: UTF-8

2. **放入文件夹**
   ```
   复制到: karuta/config/images/
   示例文件名:
   - card_spring.png
   - card_summer.jpg
   - character_A.png
   ```

3. **验证**
   - 检查图片完整性
   - 测试显示效果

### 第三步：创建卡组配置 (CSV)

#### 使用记事本或 Excel 创建 CSV 文件

**方法 1: Excel**
```
1. 打开 Excel
2. 输入表头:
   image_name | work_name | songs
3. 输入数据:
   card_spring.png | 春樱祭典 | spring1.mp3|spring2.mp3
   card_summer.png | 夏季祭 | summer.mp3
4. 保存为 "CSV (逗号分隔)" 格式
5. 移到 karuta/config/decks/ 文件夹
```

**方法 2: 记事本**
```
1. 打开记事本
2. 输入以下内容（注意: 用英文逗号和竖线）:

image_name,work_name,songs
card1.png,作品A,song1.mp3|song2.mp3
card2.png,作品B,song3.mp3
card3.png,作品C,song4.mp3|song5.mp3

3. 按 Ctrl+Shift+S （另存为）
4. 选择 "所有文件"
5. 输入文件名: deck1.csv
6. 编码改为 "UTF-8"
7. 保存在 karuta/config/decks/
```

#### CSV 格式详解

| 列 | 说明 | 示例 |
|----|------|------|
| `image_name` | 图片文件名（必须存在）| `card1.png` |
| `work_name` | 卡片显示名称 | `春樱祭典` |
| `songs` | 歌曲列表，用 \| 分隔 | `song1.mp3\|song2.mp3` |

**完整示例:**
```csv
image_name,work_name,songs
card_春樱.png,春樱祭典,春樱1.mp3|春樱2.mp3|春樱3.mp3
card_夏季.png,夏季祭,夏季开幕.mp3
card_秋枫.png,秋风诗社,秋枫1.mp3|秋枫2.mp3
card_冬雪.png,冬雪节,冬雪.mp3|雪花.mp3
```

### 第四步：调整游戏参数

编辑 `config/config.txt`:

```properties
# 音乐播放时长（秒）
min_duration=10        # 最少播放 10 秒
max_duration=30        # 最多播放 30 秒

# 中场休息
rest_music=rest.mp3    # 休息音乐文件名
failure_mode=PASS      # PASS=轮空, SKIP=弃牌

# 文件夹路径
music_folder=config/music/
images_folder=config/images/
default_deck=config/decks/deck1.csv
```

## 游戏流程

### 完整的一局游戏

```
START
  ↓
[选择卡组] → 用户从列表选择一个卡组
  ↓
[设置回合] → 用户输入回合数（如10）
  ↓
[加载资源] → 系统加载图片和音乐
  ↓
[GAME LOOP]
  ├→ [第1回合]
  │   ├→ 随机选择一张卡 (如: 春樱)
  │   ├→ 随机选择一首歌 (从该卡对应的歌中)
  │   ├→ 显示卡片图片
  │   ├→ 播放歌曲 (10-30秒)
  │   ├→ 显示 "成功/失败" 按钮
  │   └→ 等待用户操作
  │
  ├→ [用户点击 "成功"]
  │   ├→ 小时效果
  │   ├→ 该卡从场上移除
  │   ├→ 成功计数 +1
  │   └→ 下一回合
  │
  ├→ [第2-5回合] (正常进行)
  │
  ├→ [第5回合结束]
  │   ├→ 播放中场休息音乐
  │   ├→ 显示 "准备" 按钮
  │   └→ 等待用户点击
  │
  ├→ [用户点击 "准备"]
  │   └→ 继续回合 6
  │
  ├→ [第6-10回合] (正常进行)
  │
  └→ [第10回合]
      └→ 游戏结束
  ↓
[统计结果]
  显示:
  - 总回合数
  - 成功次数
  - 失败次数
  - 成功率百分比
  ↓
[返回菜单]
END
```

### 常见操作

#### 1. 监控游戏进度
- **在场卡牌** - 用绿色显示，可能再出现
- **已出局卡牌** - 用灰色显示，已完成
- **数量显示** - 实时更新卡牌数量

#### 2. 手动调整卡牌
- **移入场上** - 从 "已出局" 列表选中，点击 "➕加入场上"
- **移出场外** - 从 "在场" 列表选中，点击 "➖移出场外"

#### 3. 恢复误除
- 点击 **"⚠️清空出局"** 将所有已出局的卡恢复
- 系统会要求二次确认

#### 4. 导出状态
- 点击 **"💾导出状态"** 
- 将当前游戏状态复制到剪贴板
- 可用于记录或备份

## 常见问题

### Q1: 应用无法启动
**症状**: 双击没有反应或闪现错误窗口
**原因**: 可能未安装 Java
**解决**:
```
1. 检查 Java 是否已安装
   方法: 打开 CMD，输入 "java -version"
2. 如果显示版本号，Java 已安装
3. 如果提示命令不存在，下载安装 Java 11+
```

### Q2: 看不到卡片图片
**症状**: 游戏界面显示空白或占位符
**原因**: 图片文件缺失或路径不对
**解决**:
```
1. 检查文件是否在 config/images/ 中
2. 检查 CSV 中的文件名是否与实际文件名一致
   - ✅ 正确: card1.png 对应 config/images/card1.png
   - ❌ 错误: card1.png 对应 config/images/card_1.png
3. 确认文件格式是否正确（PNG/JPG）
```

### Q3: 无法播放音乐或无声
**症状**: 歌曲不播放或完全没有声音
**原因**: 音乐文件问题或系统音量
**解决**:
```
1. 检查系统音量是否关闭
   - Windows: 检查任务栏音量图标
   - 调试: 点击音量 100% 确保已启用
2. 检查音乐文件
   - 尝试用媒体播放器打开该文件
   - 检查文件是否完整（下载失败等）
3. 检查格式是否支持
   - 支持: MP3, WAV, FLAC, OGG, AAC
   - 尝试转换为 MP3 格式
```

### Q4: CSV 文件加载失败
**症状**: 显示 "加载卡组失败" 或具体的错误信息
**原因**: CSV 文件格式不正确
**解决**:
```
检查清单:
□ 第一行是否为: image_name,work_name,songs
□ 使用英文逗号分隔，不是中文逗号
□ 使用英文竖线 | 分隔歌曲，不是中文
□ 文件编码是否为 UTF-8（重要！）
□ 没有多余的空白行
□ 图片和音乐文件确实存在

正确示例:
image_name,work_name,songs
card.png,作品名,song1.mp3|song2.mp3

错误示例 ❌:
image_name，work_name，songs  （中文逗号）
card.png,作品名,song1.mp3┃song2.mp3  （中文竖线）
```

### Q5: 后台管理面板没有显示
**症状**: 只看到游戏窗口，没有看到管理面板
**原因**: 窗口被隐藏或窗口管理器问题
**解决**:
```
1. 查看任务栏是否有多个窗口
2. Alt + Tab 切换窗口
3. 检查屏幕外是否有窗口
4. 重启应用程序
```

## 技巧和建议

### 🎮 游戏技巧
1. **配置合理的时长** - 10-20 秒较为平衡
2. **多种音乐组合** - 让游戏更有变化性
3. **定期中场休息** - 保持参与者精力
4. **备份卡组文件** - 定期保存配置

### 📊 组织活动建议
1. **准备多个卡组** - 不同难度或主题
2. **设置合理回合数** - 小型赛: 10-15 回合，大型赛: 30+ 回合
3. **提前测试** - 确保所有文件完整
4. **后台监控** - 使用管理面板防止故障
5. **备用方案** - 准备纸质卡片作备份

### ⚡ 性能优化
1. **关闭其他程序** - 释放系统资源
2. **使用 SSD** - 加快文件读取
3. **定期清理临时文件** - 释放磁盘空间
4. **保持 Java 最新版本** - 性能和安全性
